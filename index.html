This is what real quality looks like

.cursorrules — Next.js 14 App Router + TypeScript
# Next.js 14 App Router — Vibe Coding Rules
# Stack: Next.js 14, TypeScript, Tailwind CSS, Shadcn/UI

Project Context:
- Always use App Router (/app directory), never Pages Router
- Server Components by default; add 'use client' only when required
- TypeScript strict mode enforced — no 'any' types without comment
- Use Tailwind CSS for all styling — no inline styles or CSS modules

Component Rules:
- Named exports only (no default exports for components)
- Co-locate types with component: interface Props at top of file
- Use Shadcn/UI primitives before writing custom components
- All data fetching in Server Components using async/await

API Routes:
- route.ts handlers only — no api/ directory patterns
- Always return NextResponse.json() with typed response
- Zod validation on all POST/PUT bodies — never trust input
.cursorrules — FastAPI + Python AI Agent
# FastAPI AI Agent Rules — LangChain + Pydantic v2

Architecture:
- Pydantic v2 for all models (use model_validator not validator)
- Async endpoints by default — use async def for all route handlers
- Dependency injection via FastAPI Depends() for shared resources
- LangChain LCEL chains over deprecated AgentExecutor patterns

Agent Patterns:
- StructuredTool for all custom tools with full type annotations
- Streaming responses with StreamingResponse for LLM output
- Checkpointer for agent memory (prefer PostgresSaver in prod)
- Error handling: always catch ToolException, return structured error

What exactly is a .cursorrules file? ▼
A .cursorrules file lives in your project root and gives Cursor AI persistent context about your codebase, stack conventions, and preferences. Every time Cursor generates code, it reads these rules and applies them automatically. It's the difference between Cursor generating generic boilerplate vs production-quality code that matches your exact standards.
Do I need a specific Cursor version? ▼
.cursorrules files work with Cursor 0.26+ (all current versions). They also work with Cursor's newer .cursor/rules/ directory format introduced in v0.43 — we include both formats for maximum compatibility.
